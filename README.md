# 关爱老人志愿服务平台

## 项目简介

本项目是一款面向 **老年人** 与 **志愿者** 的双端关爱服务应用，通过定时问候与智能响应机制，及时发现老年人异常情况，并在必要时将信息推送给距离最近的志愿者，实现“被动报警 + 主动关怀”的社区互助模式。

适用场景包括：

* 独居/空巢老人日常关怀
* 社区志愿服务数字化
* 街道/社区居家养老辅助系统

---

## 核心功能概览

### 老人端（被服务方）

* 手机号注册 / 登录
* 基本信息填写

  * 姓名
  * 详细地址
  * 实时或首次定位
  * 监护人联系电话
* 接收系统每日定时问候消息
* 一键或文字方式回复问候

### 志愿者端（服务方）

* 手机号注册 / 登录
* 基本信息填写

  * 姓名
  * 常驻地址
  * 实时定位（用于距离计算）
* 接收附近老人异常提醒
* 查看老人基本信息与位置

### 服务端（后台系统）

* 定时任务

  * 每日固定时间向老人端发送问候信息
* 响应监测

  * 记录老人回复时间
  * 超过 **2 小时未回复** 判定为异常
* 智能推送

  * 根据定位计算距离
  * 向最近的志愿者推送提醒信息

---

## 业务流程说明

1. 老人端完成注册并填写信息
2. 志愿者端完成注册并授权定位
3. 服务端每日定时向老人发送问候
4. 老人正常回复 → 流程结束
5. 老人 2 小时未回复 → 触发异常
6. 系统计算与老人距离最近的志愿者
7. 向志愿者端推送提醒信息

## 技术方案建议（可选实现）

### 前端

* 框架：`uni-app`
* 适配：Android / iOS / 微信小程序（可选）
* 功能模块：

  * 登录注册
  * 表单填写
  * 定位授权
  * 消息推送

### 后端

* 语言：Go / Node.js（任选）
* 框架：Gin / NestJS
* 定时任务：cron / 内置定时器
* 消息推送：

  * App 推送（如极光、个推）
  * 短信兜底（可选）

### 数据库

* MySQL：用户、角色、位置、日志
* Redis：

  * 未回复状态缓存
  * 定时任务辅助

## 合规与隐私说明

* 不对外展示老人隐私数据
* 所有通信需采用 HTTPS
