
🧓 独居老人关爱志愿者调度系统（Go 后端项目）：

项目介绍：
基于 Golang + Gin + GORM 开发的后端服务系统，用于模拟独居老人定时关怀场景。当老人长时间未回复问候信息时，系统自动匹配最近志愿者并生成告警记录，实现基础的应急调度流程。


🧱 技术栈：

Golang

Gin Web Framework

GORM ORM

MySQL

定时任务（goroutine）

RESTful API 设计

Haversine 距离算法


⚙️ 主要功能：

实现老人端、志愿者端注册接口

构建定时任务扫描机制，周期检测未回复老人

实现基于经纬度的最近志愿者匹配算法

自动生成告警记录 alert_record 并记录距离信息

实现告警防重复触发机制

支持老人回复状态更新接口

按 handler / service / task 分层设计后端结构


🚀 项目亮点：

设计自动调度流程，模拟真实养老应急响应场景

实现服务层拆分，提高业务逻辑复用性

使用 goroutine 实现后台定时扫描任务

基于 Haversine 公式实现地理距离计算

构建完整告警生命周期（创建 → 记录 → 防重复）

RESTful API 规范化设计


📈 项目职责：

数据库结构设计与建表

后端 API 架构设计与实现

定时任务模块开发

距离匹配算法实现

告警自动化流程设计

Postman 接口测试


⭐ 项目总结：

该项目模拟真实养老服务场景，从用户注册、定时任务调度、
自动匹配到告警生成，实现了完整的自动化应急处理流程，
提升了对 Go Web 后端架构与业务拆分的理解。
